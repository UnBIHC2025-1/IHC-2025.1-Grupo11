    Progresso

    Desenvolvimento

    Para garantir que nossos produtos digitais sejam acessíveis a todos, é fundamental que a equipe de desenvolvimento esteja bem preparada. Isso envolve saber como implementar recursos de acessibilidade, desde a fase de design até a codificação, e como verificar sua eficácia.

    É crucial que a equipe tenha familiaridade com as ferramentas disponíveis no mercado para testes e padronizações de acessibilidade. Além disso, uma compreensão aprofundada de como pessoas com deficiência interagem com websites e aplicativos é essencial para criar experiências verdadeiramente inclusivas.

    Testes

    • Foram realizados testes manuais de acessibilidade? [NBR]
    • Foram realizados testes com pessoas com deficiência? [NBR]
    • Todos os elementos da página são navegáveis apenas com teclado? [12] [NBR]
    • Foram utilizados leitores de tela (ex.: NVDA, VoiceOver, JAWS) nos testes? [1] [NBR]
    • A navegação em dispositivos móveis foi testada para garantir acessibilidade? [14] [NBR]
    • Foram realizados testes de contraste de cores? [5] [NBR]
    • O site foi testado em diferentes navegadores e sistemas operacionais? [NBR]

    Código e Estrutura

    Imagens

    • Todas as imagens relevantes possuem texto alternativo (alt) descritivo? [1] [NBR]
    • As imagens decorativas possuem alt="" ou são inseridas via CSS? [1] [NBR]
    • Nenhuma informação relevante está embutida dentro de imagens? [1] [NBR]
    • Descrições longas são fornecidas no alt quando necessário (ex.: gráficos, imagens com texto)? [1] [NBR]
    • Imagens complexas possuem descrição longa em local apropriado? [1] [NBR]

    Vídeos e Mídia

    • Todos os vídeos possuem legendas embutidas ou closed caption? [2] [NBR]
    • Existe transcrição textual para conteúdos em áudio? [1] [NBR]
    • Foi evitado o uso de conteúdos com flashes que possam causar convulsões? [3] [NBR]
    • O autoplay de mídias está desabilitado por padrão? [11] [NBR]
    • Possui pausa para todas as mídias? [12] [NBR]
    • Possui type para botões e entradas? [4] [NBR]
    • Vídeos possuem audiodescrição? [2] [NBR]
    • Existe alternativa visual para áudios importantes? [2] [NBR]

    Controles e Navegação

    • Todos os links possuem href correto? [4] [NBR]
    • Links que abrem em nova guia ou nova janela estão claramente sinalizados? [8] [NBR]
    • Todos os botões possuem funções semânticas corretas (type="button", type="submit")? [4] [NBR]
    • Foi implementada navegação por skip-link (atalho para conteúdo principal)? [7] [NBR]
    • O foco do teclado é visível e lógico em toda a navegação? [6] [NBR]
    • Links que abrem em uma nova guia ou janela estão identificados e indicados? [8] [NBR]
    • Adicionar estados de foco em campos de entrada, botões, e elementos interativos. [6] [NBR]
    • A navegação pode ser feita por teclado sem armadilhas de foco? [12] [NBR]
    • O primeiro elemento focável permite ir diretamente para o conteúdo principal? [7] [NBR]

    Formulários

    • Todos os campos possuem label corretamente associados? [9] [NBR]
    • Foram utilizados agrupadores como <fieldset> e <legend> quando necessário? [4] [NBR]
    • Campos possuem autocomplete configurado? [10] [NBR]
    • Campos de entrada possuem aria-describedby? [NBR]
    • As mensagens de erro estão visíveis, com bom contraste e próximas aos campos? [NBR]
    • Há descrição clara para os erros e instruções de preenchimento? [5] [NBR]
    • O formulário pode ser enviado usando apenas o teclado? [12] [NBR]
    • Campos obrigatórios estão claramente indicados? [NBR]

    Semântica e texto

    Semântica

    • O HTML possui marcação semântica (uso de header, nav, main, footer, article, etc.)? [4] [NBR]
    • A hierarquia dos títulos (h1 a h6) segue uma ordem lógica e progressiva? [16] [NBR]
    • Os rótulos, botões e links possuem textos claros e objetivos? [4] [NBR]
    • Foram evitados nomes genéricos como "Clique aqui"? [4] [NBR]
    • Tabelas são usadas apenas para dados tabulares, não para layout? [4] [NBR]

    Texto e Leitura

    • Nenhum texto importante está embutido em imagens? [1] [NBR]
    • É possível aumentar o zoom da página até 200% sem perda de conteúdo ou funcionalidade? [15] [NBR]
    • O tamanho da fonte é relativo (ex.: em, rem) e não fixo (ex.: px)? [3] [NBR]
    • O texto possui contraste suficiente com o fundo? [5] [NBR]
    • O texto não está em caixa alta, condensado ou justificado? [NBR]

    Teclado e Acessibilidade Operacional

    • Toda a navegação da página pode ser feita usando apenas teclado? [12] [NBR]
    • O primeiro elemento focável permite ir diretamente para o conteúdo principal? [7] [NBR]
    • Foram evitadas armadilhas de foco (ex.: foco que não sai de um modal)? [12] [NBR]
    • Existe suporte a atalhos de teclado (TAB, ESC, etc.)? [12] [NBR]
    • O foco é sempre visível (ex.: via CSS: :focus, :hover)? [6] [NBR]
    • Quando se tem o mouseover é permitido o uso de teclado? [NBR]

    Componentes e Elementos especiais

    Título

    • A hierarquia de conteúdo da página é definida por sua lógica, não pelo tamanho do texto? [16] [NBR]
    • Use elementos de título h1 h2 h3 para apresentar o conteúdo. [16] [NBR]
    • Não pular níveis lógicos. [16] [NBR]
    • Toda página contém um título h1 descrevendo a página. [16] [17] [NBR]

    Tabela

    • Elementos em formato de tabela utilizam table? [4] [NBR]
    • Todas as tabelas usam thead, tbody, th e td corretamente? [14] [NBR]
    • Foram aplicados scope="col" ou scope="row" quando necessário? [14] [NBR]
    • Cada tabela possui uma legenda descritiva usando <caption>? [16] [NBR]
    • Tabelas não possuem células mescladas sem necessidade? [NBR]

    Modais e Pop-ups

    • Modais são fáceis de fechar (ex.: com tecla ESC)? [17] [NBR]
    • Não é possível abrir um modal dentro de outro modal (sem controle)? [17] [NBR]
    • Modais não bloqueiam toda a tela (evitar fullscreen)? [17] [NBR]
    • Existe foco correto no conteúdo quando o modal é aberto? [17] [NBR]
    • A interação é uma tarefa simples? [17] [NBR]

    Dispositivos Móveis e tocáveis

    • O site funciona corretamente em qualquer orientação (paisagem ou retrato)? [13] [14] [NBR]
    • A rolagem horizontal foi evitada? [15] [NBR]
    • Botões e links possuem área de toque mínima de 44px por 44px (WCAG 2.5.7)? [17] [NBR]
    • Há espaçamento suficiente entre elementos interativos? [7] [NBR]
    • O layout é responsivo para diferentes tamanhos de tela? [NBR]

    Ferramentas e Funcionalidades extras

    • É possível pausar, parar ou ocultar conteúdo em movimento? [11] [NBR]
    • Breadcrumbs foram implementados para indicar a localização no site? [NBR]
    • Existe uma página de acessibilidade ou FAQ acessível? [NBR]
    • Áreas clicáveis seguem o tamanho mínimo de acessibilidade? [17] [NBR]
    • Captchas oferecem alternativas acessíveis (ex.: captcha de áudio ou lógica simples)? [NBR]
    • Existe um campo de busca disponível e acessível? [NBR]
